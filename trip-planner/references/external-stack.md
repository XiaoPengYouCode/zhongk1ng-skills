# 外部能力栈

这个文件记录当前可复用的公开项目，供 `trip-planner` skill 在开工前快速判断能不能站在现成积木上。

更新时间：`2026-04-26`

## 火车票

### `12306-mcp`

- 链接：<https://github.com/Joooook/12306-mcp>
- 适合做什么：查车次、查余票、查中转、登录、下单
- 备注：README 里明确列出查询购票信息、车站信息、经停信息、中转方案、乘客管理、登录、下单相关能力。适合作为火车票主通道。

### `12306 Train Assistant`

- 链接：<https://termo.ai/skills/12306-train-assistant>
- 适合做什么：通过浏览器自动化走 12306 官网登录、查票、填乘车人、提交订单
- 备注：适合没有本地 MCP、但需要完成真实官网流程的场景。

## 机票

### `Tuniu Flight`

- 链接：<https://termo.ai/skills/tuniu-flight>
- 适合做什么：查机票、看价格日历、创建订单、查询订单、取消订单
- 备注：页面公开了 `https://openapi.tuniu.cn/mcp/flight` 和 `TUNIU_API_KEY` 依赖。适合把机票查询和预订接成标准 MCP 流程。
- 本地补充：当前环境里实际命令名是 `tuniu`，不是 `tuniu-cli`。机票链路优先用 `searchLowestPriceFlight`，候选入围后再用 `multiCabinDetails` 拿 `cabinPriceId` 做下单前核验。

### 实时订票页兜底

- 链接示例：`Trip.com` 路由页、航司落地页、可直接进入下单的 OTA 搜索结果页
- 适合做什么：在没有 flight MCP key 的环境里，验证某一天是否真有某个航班、价格是多少、是否能进入预订
- 备注：只能把带日期、航班卡片、价格和 `预订` 入口的页面视作有效信源。均价页、营销页、历史统计页无效。若票价页面未拆税费，必须继续推进到下单前页面。

### 航旅纵横机票验真

- 适合做什么：在已经拿到候选机票后，辅助核对真实票价和税费
- 备注：适合作为验真工具，不适合作为唯一搜索来源。新闻报道提到它提供"机票验证"能力，可用于核对购票信息中的真实价格与税费。

## 通用兜底

### `browser-use`

- 链接：<https://github.com/browser-use/browser-use>
- 适合做什么：没有专用订票 skill 时，用通用浏览器自动化去官方站或主流订票站抓实时结果、推进到下单前流程
- 备注：适合作为最后兜底，不适合替代专用订票接口做大规模比价。

## 使用策略

按这条顺序尝试：

1. 火车：`12306-mcp` -> `12306 Train Assistant` -> 浏览器自动化
2. 机票：`Tuniu Flight` -> 浏览器自动化

只要拿到实时价格和可订状态，就立刻进入统一成本模型，不要在工具层反复横跳。

## 12306 实时校验规则

火车推荐必须以 `12306` 官方实时余票为准。

最低要求：

1. 先访问 `https://kyfw.12306.cn/otn/leftTicket/init` 取 cookie
2. 再访问 `https://kyfw.12306.cn/otn/leftTicket/query` 或官方返回指示的 `queryG`
3. 同时检查：
   - 查询结果里真的有这班车
   - 结果处于可预订状态
   - 用户目标坐席不是 `无`

禁止的偷懒做法：

- 只看第三方时刻表页就下结论
- 只看列车是否存在，不看这天是否开行
- 只看"理论票价"，不看当前是否还有票

## 节假日与绕行补充

真实规划时，额外做两步：

1. 查官方节假日安排，确认哪些半天其实不算请假
2. 查夜间动卧到大枢纽再短接的组合，尤其是 `深圳北 -> 上海虹桥` 这一类长途睡一晚、清晨转常州的方案

周边机场默认纳入比价：

- 无锡 `WUX`
- 扬州泰州 `YTY`
- 南京 `NKG`
- 南昌 `KHN`
- 上海虹桥 `SHA`
- 上海浦东 `PVG`

周边车站默认纳入比价：

- 上海
- 上海虹桥
- 常州北
- 武进
- 金坛
- 无锡
- 苏州
- 南京南
- 马鞍山东
- 蚌埠南
- 南昌西
